Linux file system hierarchy.

Entenda o que armazena e para que serve cada diretorio.

Postado por metavops 05 de junho de 2021.

Como funciona o linux file system.

Muitas pessoas quando iniciam no linux costuman fircar um pouco perdidas por conta da organização dos diretorios, pois o linux não tem um disco " C: " e os seus programas não vão ficar salvos em uma pasta chamada "Arquivos de Programas" ou "Arquivos de Programas(x86)", a forma em que os diretorios são organizados no linux é um pouco diferente.

Nesse post irei te informar oque cada diretorio armazena, e para que eles servem no sistema.

/
O / é o nosso diretorio raiz, é dele onde sair todos os arquivos e diretorios do nosso sistema, seria mais ou menos como o disco "c:" no windows.

/bin
Armazena binarios executaveis que pode ser executado por qualquer usuario do sistema, trata-se do diretorio que armazena os comando basicos do sistema.

/sbin
Armazena binarios executaveis que pode ser executado somente por administraores, pois os comandos armazenados nele são para manutenção do sistema.

/boot
Diretorio onde armazena os arquivos de inicialização do sistema, seja seu sistema com base em EFI ou MBR.

/dev
Armazena informações sobre discos HD's, SSD's, disquetes, divers de CD/DVD e entre outros dispositivos conectados é maquina.

/etc
Diretorio onde fica armazenado configurações de serviços e aplicações, nelo podemos configurar repositorios, configurações de rede e etc.

/home
Diretorio na qual fica localizado arquivos dos usuarios do sistema, como se fosse a "área de trabalho" dos usuarios no linux.

/midia
Fica armazenado informaçẽs sobre midias removiveis, como por exemplo CD's, pendrives, cartões de memoria e etc.

/mnt
Diretorio utilizado por administradores para montar sistemas de arquivos temporarios e externos.

/opt
Aplicativos que não são essenciais para o funcionamento do sistema vão para esse diretorio, como por exemplo: google, skype, teamviewer, hamachi e etc.

/srv
Armazena informações de seviços ativos no sistema.

/lib
Armazena informações de blibliotecas utilizadas pelo sistema e aplicações.

/proc
Nesse diretorio podemos encontrar informações de recusos e processos do sistema e informações basicas como por exemplo, quantos cores nosso processador tem e o tempo em que a maquina esta ligada.

/root
Diretorio do usuario raiz do sistema.

/tmp
Como o proprio nome dá á entender é um diretorio temporario que armazena arquivos e impede que esses arquivos ocupem o diretorio / impedindo assim travamentos no sistema.

/usr
Contem programas e bibliotecas, codigos fontes, e documentação, por exemplo ao instalarmos um programa á partir de seu cofigo fonte, após a instalação os arquivos do programa ficarão nesse diretorio.

/var
Diretorio onde armazena logs, filas de e-mails, arquivos de banco de dados, como é um diretorio que esta sempre em expansão/alteração, é recomendado que vocẽ separe uma boa partição para ele.
