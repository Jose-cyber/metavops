O que é git flow e como utilizar ?.

Aprenda a como utilizar o git flow e gerenciar suas branches e features de maneira inteligente.

Postado por metavops 29 de agosto de 2021.

O que é o git flow ?

O Gitflow é um modelo alternativo de ramificação do Git que consiste no uso de ramificações de recursos e várias ramificações primárias. isso permite ter uma melhor organização e versionamento do codigo.
Se compararmos o git flow com o desenvolvimento baseado em troncos(Trunk based development), podemos concluir que o git possui mais ramificações

Features no git flow

Features se trata-se de novas funcionalidades na qual esta sendo adicionadas a seu projeto/produto, o git flow trata essas features em forma de branches que são criadas a partir da branch develop, dessa forma cada desenvolvedor ou equipe pode trabalhar em uma feature especifica e assim que terminar o desenvolvimento da nova funcionalidade, essa branch é encerrada e o conteudo dela é mergiado para a branch develop.

Tipos de branches

Utilizando o modelo de ramificação do git flow, você pode ter varias branches, e cada tipo tem uma função especifica:

Master/Main: Utilizamos essa branch para armazenar o codigo para o ambiente de produção, ela pode ter o nome de master ou main.
Hotfix: São branches que podem ser criadas com base no conteudo da master/main e são para trobleshoot ou resolução de bugs em produção.
Releases branches: As realeases branches, são branches de "homologação", você pode manter mais de uma branch de release com versões diferentes, e ao finaliza-la ocorre o merge para as branch master/main.
Develop: Branch de desenvolvimento.
Feature branches: São branches de novas funcionalidades que posteriormente será feito o merge para a branch develop ao ser finalizada.

Dessa forma podemos concluir que sempre teremos duas branches principais, que seria a develop e a master ou main, o que simplifica caso exista algum fluxo de CI/CD, minimiza problemas de conflitos, e permite uma melhor organização das features que estão em desenvolvimento.

Git flow na pratica ?

Conclusão.

O git flow é uma excelente ferramenta e bastante importante para manter padrões durante o desenvolvimento de software, trabalhar e organizar melhores as features, minimizar merge conflicts, e otimo para caso queira integrar essa pratica a um CI/CD.
